# Data artifacts

Generated by Phase 0 scripts. Imported by the Replit Lenny Tank app.

- `guests/{slug}.json` — character sheets (50 guests). See `index.ts:CharacterSheet`.
- `chunks/{slug}.json` — embedded transcript chunks (~500 words each). See `index.ts:Chunk`.
- `scenarios.json` — 25 curated scenario cards across 5 buckets. See `index.ts:ScenarioDeck`.

**Embedding model:** chunks in this folder have **Ollama `nomic-embed-text` (768-dim)** embeddings, generated locally during the original Phase 0 work. These are **NOT used by the deployed app** — they exist as a historical artifact / local-verification dataset.

**Runtime embeddings** live in the Replit project at `artifacts/web/data-runtime/chunks/*.json`, generated with **OpenAI `text-embedding-3-small` (1536-dim)**. See `docs/decisions.md` §1 for the rationale.

## Regenerating

If you need to re-run anything locally:

```bash
npx tsx scripts/chunk-transcripts.ts          # re-chunk transcripts (no API key needed)
npx tsx scripts/embed-chunks.ts               # local Ollama re-embed (requires `ollama serve` — local-only artifact)
npx tsx scripts/build-character-sheets.ts     # requires ANTHROPIC_API_KEY in .env
```

For the runtime embeddings inside Replit:
```bash
npx tsx artifacts/web/scripts/reembed-with-openai.ts   # requires OPENAI_API_KEY in Replit Secrets
```

Each script is idempotent — re-running skips already-completed work.
